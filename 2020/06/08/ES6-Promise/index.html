ES6 Promise

这里仅分享 ES6 中的 Promise 的特性和用法，只浅显回答 是什么和 怎么用的问题。 如果需要深入了解原理，请参考另外一篇。

Promise 是什么

以前，对于 JavaScript 中传统的异步编程，一般会有两种解决方案：回调函数，发送事件。如果流程和数据传递比较复杂，回调可以说是相当冗长繁复了。而 Promise ，就是为了解决异步编程的问题，较之于以前的方案更加强大优雅。

本质上来说，Promise 是一个对象，持有异步操作的状态，用来表示异步操作的不同结果。遵循两个原则：

  • 对象只有（如下）三种状态，状态由操作的结果决定，且没有其他方式更改


    • pending 进行中
    • fulfilled 已完成
    • rejected 已失败

  • 状态改变只有两种情况：


    • 从 pending 变为 fulfilled

    • 从 pending 变为 rejected


可以简单理解为 Promise 中的异步操作，只能从进行到结束，而且一旦结束，结果就不会再变。不能暂停，不能取消，也不能改变已经产生的结果。

区别于事件监听机制的错过事件后就得不到结果的特性，如果 Promise 异步操作已经结束，状态已经改变，那在此之后的任何时候依然是可以通过对 Promise 添加回调函数获取到这个结果的。

如果 Promise 异步操作中，状态还处于 pending 未改变时，外部对 Promise 的具体阶段是不可见的，也不能手动取消该该异步操作。

Promise 怎么用

知道了 Promise 会有三种状态，处理异步操作，那具体咋用呢？

resolve reject 传递异步操作结果

通常来说，resolve 用于传递异步操作成功的结果，而 reject 用于传递失败时候的结果或者出现的异常。当调用 resolve 传递异步操作结果的时候，Promise 的状态就从 pending (进行中) 改变成了 fulfilled (已完成)，对应的，调用 reject 传递结果时，Promise 的状态从 pending (进行中) 变成了 resolved (已失败)。

不过这也只是约定俗成，你仍然可以通过 reject 传递成功的结果而 resolve 传递失败的结果，但是这样会影响到正常取值，错误及异常处理的正常流程

then() 方法指定回调

Promise 对象实例生成，执行对应的异步操作获取到结果后，可以用 then 方法指定不同状态的回调。

then 方法接受两个回调函数作为参数，第一个回调函数处理 resolved 状态，在 Promise 状态变为 resolved 时调用，并接受 resolve 函数传递的值为参数。第二个函数用于处理 rejected 状态，是可选的，在 Promise 对象变为 rejected 时调用，同样可以接受 reject 函数传递的值为参数

其他 Promise API 使用

上面是 Promise 的一些基础使用，已经可以进行简单的异步任务的处理，但是如果是多个异步任务或者其他复杂的情形，还需要了解其他 Promise API。

总结：

then 方法可以链式调用。

前一个 then 方法的回调函数 return 普通数据（非 Promise 对象）时，后一个 then 只会调用第一个回调函数， 且回调函数的参数就是该普通数据。

前一个 then 方法的回调函数 return 另一个 Promise 对象的时，后一个 then 方法等待该 Promise 对象的状态改变，再执行对应的回调函数（resolved 对应第一个回调，rejected 对应第二个回调）。

如果异步操作结果失败，你也可以在 Promise 中 throw error, 这和 reject 效果一样，会使 Promise 变为 rejected 状态。抛出的错误可以被 then 方法的第二个回调或者 catch 捕获。

不过如果在 resolve 语句之后抛出错误，是无效的 。可以理解为这都是引起 Promise 状态改变的操作， resolve 之后状态已经改变为 resolved，不能再抛出异常变为 rejected。

Promise 对象的错误如果不被捕获，就是一直向后传递，直到被 then 方法的第二个回调或者 catch 方法捕获到。同时一旦 Promise 错误已经被捕获，再在后面添加 then 方法或者 catch 就捕获不到了。

上面的 catch 方法是捕获不到错误的，因为错误已经被 then 方法的第二个回调捕获。

一般来说，使用更多的是用 catch 来捕获错误。相比于 then 方法的第二个回调，catch 除了可以捕获到 Promise 异步操作的错误，也可以捕获到 then 方法第一个回调的错误。

catch 方法返回的还是一个 Promise 对象，后面可以继续正常调用 then 方法。此时如果 catch 方法没有捕获到错误，则会直接跳过 catch , 执行后面的 then 方法。

catch 方法还可以继续抛出错误，再由后面的 catch 方法捕获。

总结：

catch 和 then 方法的第二个回调都可以捕获错误。

Promise 对象的错误会向后传递直到被捕获，同一个错误一旦被捕获就不能再次被捕获。

常用的 promise.then().catch() 中， catch 可以捕获对应 then 中的错误，因而更常用。

catch 方法的返回值依然是 Promise，可以继续链式调用 then 和 catch。
