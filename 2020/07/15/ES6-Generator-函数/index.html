Generator 函数

初识 Generator 函数

Generator 也是 ES6 提供的一个异步编程解决方案（译为生成器，可以理解为生成多个可迭代值的函数）。函数内部封装了多个状态，调用时可以得到一个迭代器对象，通过迭代器对象，就能遍历出 Generator 函数的每一个值。

关于 Iterator 迭代器和迭代协议可以参见这篇：ES6  系列  Iterator

看一个简单的例子 🌰：

函数体内部使用yield语句（后面介绍yield语句）生成值，返回的迭代器对象是符合可迭代协议和迭代器协议的，所以可以持续调用next方法得到迭代值， return 函数并不需要我们显式指定。

Generator 函数调用后并不立马执行，而是返回一个指针对象(迭代器对象)，指向的是内部状态。只有调用迭代器对象的next方法，指针才会移到下一个状态。第一次调用next方法，会从函数头部开始执行，直到遇到一个yield表达式、return语句或者throw语句，而再次调用的时候会从上次yield语句的地方开始执行。

  • yield 。暂停 Generator 函数并返回得到生成器新值，下一次next调用会从上一次的yield表达式位置开始执行。
  • throw 。可以从生成器中抛出异常，生成器会完全停止，需要在调用处捕获。
  • return 语句。生成器的执行结束，将迭代结果返回给调用者，返回的done为true，视为迭代完成。
  • 到达 Generator 函数末尾，生成器正常执行技结束，得到value为undefined，且done为true。

除了可以暂停和恢复执行，还需要注意 Generator 函数一些形式上特征：

  • 函数名上需要带星号 * (星号可以写在function关键字和函数名之间的任意位置，推荐是紧跟function关键字)
  • 函数体内部使用yield表达式生成值

另外，因为 Generator 函数执行返回的是一个迭代器对象，那也就是可以作为对象的[Symbol.iterator]属性，使对象可迭代。

如上对obj使用扩展运算符迭代得到的是[ 1, 2 ]，已经按照我们预设的方式进行了迭代。

yield 表达式

可以认为，Generator 函数是可暂停执行的函数，yield语句可以用来暂停执行生成值，next方法可以恢复执行。且yield表达式只能用于 Generator 函数中。yield表达式运行逻辑是这样的：

  1. 执行到yield，Generator 函数会暂停，并将后面的值作为迭代时的 value 值。

  2. 只有调用next方法，Generator 函数才会继续执行，如果再遇到yield，则同样暂停并赋予新的迭代时的 value 值。如果没有yield表达式，则继续执行，后面也没有return语句的话，就会执行完成，此时的 value 为 undefined。有return语句的话，return后面的值会作为value值，且函数结束。

如果yiled后面跟的也是一个表达式，该表达式只有当通过调用next方法运行到yield语句时，才会执行。

yield表达式可以用在另一个表达式中，但必须放在圆括号中。yield表达式还可以用作函数参数，但只是在yield表达式执行完之后继续调用next该函数才会执行。

第一次调用next的时候，只会得到1，第二次调用next的时候，才会输出Hi，此时函数执行完，value 是undefined。

yield 表达式的返回值

yield表达式本身是没有返回值的(undefined)，而next方法在调用时如果带一个参数时，该参数就会被当作上一个yield表达式的返回值。

yield* 表达式

如果在 Generator 函数中需要调用另一个 Generator 函数或者其他可迭代对象，就需要用到yield*表达式，yield*表达式可以产生迭代对象返回的的每个值，yield*本身的返回值是当迭代器结束时返回的值。

如上，gen2中包含了对gen1()和字符串xy，并对这两个数据使用了yield*命令，实际上相当于将他们依次迭代，并将迭代结果再使用yield，故对gen2()对象遍历，也得到了gen1()和字符串的迭代结果。另外，最后一次迭代的结果，会被作为yield*的返回值，如上的[result] C，如果将gen1中的return "C"替换成yield "C"，那么resuilt就是 undefined。

Generator.prototype 的 next,throw,return 方法

Generator 函数返回的迭代器对象，都有三个重要的方法，next,throw，return方法。next方法已经介绍过，可以恢复函数执行，并可以向next方法传值作为上一个yield语句的返回值。关于throw和return:

  • throw 可以用于在函数体外抛出错误，并且在 Generator 函数体内捕获。这里的 throw 是通过调用 Generator 函数得到的迭代器调用的，需要区别于全局的 throw 命令。

next使 Generator 函数执行到yield，a 错误被 Generator 函数内部捕获，Generator 函数执行完。下一个 b 错误就只能被外部的try-catch语句捕获，得到 b。而外部函数的try-catch中，抛出 b 错误且被自身的catch捕获到，后面的throw("c")不会被执行。

如果 Generator 函数内部没有指定try-catch语句，则抛出的错误只能被外部的try-catch捕获，如果外部没有try-catch，那么函数会直接报错，中断执行。另外，如果迭代器对象没有调用过next方法，是不能执行throw方法的，因为next方法执行时，才真正执行 Generator 函数，此时的 throw方法会将错误直接抛出到函数外部。

另外，如果 Generator 中try-catch语句后面还有yield，那么throw错误被 Generator 函数内捕获之后，还会附带执行一次next，触发try-catch后面的yield语句。如果 Generator 执行时抛出的错误没有被内部捕获，那么 Generator 函数就不会再执行下去。再执行next得到的是{value:undefined,done:true}。

  • return 可以返回给定的参数值，将指定的参数作为value，如果不指定参数则value是undefined。但是return一旦执行，Generator 函数的迭代就算作终止了，done为true。

如果 Generator 函数内还有try-finally语句块，且在执行try语句时调用了return,那么 Generator 函数会直接进入finally语句，finally语句执行完之后，才会通过return语句使 Generator 结束。

X 先于 C 和 D 被迭代，因为return导致finally执行。

next,throw,return三个方法都可以让 Generator 函数恢复执行。next方法可以传值并作为yield表达式的值，throw抛出错误，需要被try-catch在函数内部或外部捕获。return则终止 Generator 函数执行，可设定迭代value值。

Generator 函数实现异步

到这里，Generator 函数的使用只是和Iterator迭代相关联，还没有真正进入他的核心使用方法**–异步编程**。在进入 Generator 函数的异步编程应用之前，需要简单了解一下Thunk函数。

Thunk 函数

解释：在对函数调用传参时，如果参数是一个计算结果，则将该计算过程封装到一个临时函数里面，再将临时函数传入到要调用的函数中。这个临时函数就是Thunk函数。

在 JavaScript 中Thunk函数的含义略有不同，JavaScript 的Thunk函数则是将多参数函数替换成一个只接受回调函数的参数。

Generator 函数通过 Thunk 封装异步操作

已经知道，yield可以暂停 Generator 函数执行，实际上是交出了程序的执行权，可以通过调用next方法使 Generator 函数再次获取执行权，而调用next方法可以放到回调中进行。 下面在用 Generator 函数实现两个异步任务，并且第二个异步任务使用第一个异步的结果来执行（等待第一个异步结束获取到结果再将其作为第二个异步任务的参数）。

传入的值是3，两秒后得到4，第一个异步流程执行完。而后第一个异步的结果4又被当做参数传到第二个异步任务中，所以得到5。这里异步就按照预期执行了。

上面第一次调用g.next()的时候，程序执行权交给 Generator 函数，并执行到了yield asyncTask(3)，此时交出执行权。和上面的例子一样，asyncTask(3)的执行结果是一个 Thunk 函数，并需要接受一个回调函数作为参数。 而yield命令使该函数也是挂在返回的{value:..,done:...}对象的 value 属性上的，所以可以通过result1的 value 获取到该 Thunk 函数。

result1.value是一个 Thunk 函数，就需要传入一个回调函数，上面传入的是(data1) => {g.next(data1)...，回调函数中执行了g.next方法, Generator 函数再次获取了执行权。data1作为参数传入了next方法，故a1的值就是data1(yield 表达式的返回值)。同样，g.next(data1).value也是一个 Thunk 函数，继续为其添加回调就可以进行后面的操作，这里就已经通过 Generator 函数将两个异步过程"串联"了起来。

关键点：通过将异步操作封装到 Thunk 中，在 Generator 中通过yield关键字，回调函数中执行next方法控制执行。

Generator 函数的自动执行
